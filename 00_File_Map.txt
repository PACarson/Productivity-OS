=====================================
COMMON EDIT SCENARIOS
=====================================

"Add new Telegram command"
  1. 06_Parser.gs    — add parseXxx()
  2. 01_Config.gs    — add EVENT.NEW_EVENT
  3. 07_EventEngine  — add case in switch
  4. 08_Router.gs    — add route (check order! P5 in Constitution)
  5. XX_Service.gs   — add handler function + update return {}
  6. 00_File_Map.gs  — update EVENT→SERVICE mapping
  7. 00_Project_State.gs — update command reference

"Change maintenance interval for a bike"
  → Edit Maintenance_Profile sheet directly (col 3 = KM_Interval)
  → MaintenanceService.record() and MaintenanceIntelligence both
     read from this sheet first (CR11)

"Add new Google Sheet"
  1. 01_Config.gs    — add SHEETS.NAME + COL.NAME
  2. 15_Setup.gs     — add to SCHEMA array + seed function
  3. Run setupRiderOS()

"Fix a bug in shift logic"
  → 09_DailyService.gs + 05_StateService.gs

"Fix maintenance ETA wrong"
  → 29_MaintenanceIntelligence.gs (_calcAllStatus etaDays logic)
  → 32_UserProfile.gs (getAvgDailyKm source data)

"Fix reward calculation"
  → 24_RewardService.gs (dashboard, _getWeekPeakTotal, _parseTiers)

"Fix booking count wrong"
  → 27_BookingEngine.gs (_getBookingStats, _getCriticalMissingSlots)

"Telegram not responding"
  → 14_Main.gs (check doPost has NO return statement)
  → Run registerWebhook() or checkWebhook()

"NaN:31 AM showing in shift message"
  → 04_TruthEngine.gs formatTimeHuman() — should handle Date objects
  → Verify START_TIME column is TIME-formatted in Daily sheet

"Duplicate RIDER BRIEF on /stop"
  → 29_MaintenanceIntelligence.gs scan() — check clearPendingByBike() call
  → 31_ReminderQueue.gs — check DEDUP_HOURS (should be 6)

"1899 date showing"
  → Use TruthEngine.formatTimeHuman() for time strings
  → Never: new Date('HH:mm:ss')
  → Never: new Date(`${date}T${time}`) — parses as UTC

"Wrong interval after /service"
  → 20_MaintenanceService.gs _getMaintenanceProfile() — reads Maintenance_Profile
  → Verify Maintenance_Profile sheet has correct KM_Interval for bike+category

"Cross-bike KM contamination"
  → 05_StateService.gs (ensure bike param passed to getActiveShift)
  → 09_DailyService.gs (ensure curBike used in stopShift)

"Weekend reward shows wrong days left"
  → 24_RewardService.gs dashboard():
     daysLeft = dayNum >= 5 ? 8 - dayNum : 4 - dayNum + 1
